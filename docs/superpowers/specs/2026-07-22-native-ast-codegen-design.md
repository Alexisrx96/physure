# Design Spec: Native AST Code Generator for Physure Core

- **Author**: Irvin Torres & Antigravity
- **Date**: 2026-07-22
- **Status**: Approved

## 1. Goal

Migrate the transpilation engines (Rust, Python, Java) from regex-based text parsing in the VS Code extension (`vsc_physure`) directly into the Rust native core (`physure-core/src/codegen/`). The new transpiler will traverse the formal PHS Abstract Syntax Tree (`Statement` and `Expr`) generated by `parse_phs()`.

---

## 2. Architecture & Modules

The new `codegen` module will be located inside `physure-core/src/codegen/`:

```
physure-core/src/
  ├── codegen/
  │   ├── mod.rs        # Main entry point, Target enum, CodeGenerator trait
  │   ├── rust.rs       # Native Rust Code Generator (0 PhsInterpreter)
  │   ├── python.rs     # Native Python / Pint Code Generator
  │   └── java.rs       # Native Java 8 & 21 Code Generator
  └── lib.rs            # Re-exports pub fn transpile(target: Target, phs_code: &str) -> PhysureResult<String>
```

---

## 3. Core API Specification

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Target {
    Rust,
    Python,
    Java,
}

pub trait CodeGenerator {
    fn generate_program(&mut self, statements: &[Statement]) -> PhysureResult<String>;
    fn generate_statement(&mut self, stmt: &Statement) -> PhysureResult<String>;
    fn generate_expr(&self, expr: &Expr) -> PhysureResult<String>;
}

/// Primary public entry point for native PHS transpilation.
pub fn transpile(target: Target, phs_code: &str) -> PhysureResult<String> {
    let statements = crate::phs::parse_phs(phs_code)?;
    match target {
        Target::Rust => rust::RustCodeGenerator::new().generate_program(&statements),
        Target::Python => python::PythonCodeGenerator::new().generate_program(&statements),
        Target::Java => java::JavaCodeGenerator::new().generate_program(&statements),
    }
}
```

## 4. AST Node Handling

### 4.1 `Statement` Handling
- **`Statement::Assign { name, expr }`**:
  - **Rust**: `let mut name = <expr_rust>;`
  - **Python**: `name = <expr_py>`
  - **Java**: `Quantity name = <expr_java>;`
- **`Statement::FnDef { name, params, body }`**:
  - **Rust**: `let name = |<params>| -> Result<Quantity, Box<dyn std::error::Error>> { ... };`
  - **Python**: `def name(<params>): ...`
  - **Java**: `PhyFunction name = PhyFunction.of("<name>", ...);`
- **`Statement::DisplayText(text)`**:
  - Parses `{var}` and `{var => target_unit}` placeholders.
  - **Rust**: `println!("...", var1, var2.to("unit")?);`
  - **Python**: `print(f"...")`
  - **Java**: `System.out.println(String.format("...", var1, var2.to("unit")));`

### 4.2 `Expr` Handling
- **`Expr::Number(val)`**: `Quantity::new(val, "")?`
- **`Expr::Ident(name)`**: `name.clone()`
- **`Expr::Uncertainty { val, unc }`**: `Quantity::new_scalar(val, unc, ...)`
- **`Expr::Binary { op, left, right }`**: `<left>.<op_method>(&<right>)?`
- **`Expr::Convert { expr, target_unit }`**: `<expr>.to("<target_unit>")?`
- **`Expr::Ternary { cond, then_expr, else_expr }`**: Native conditional logic.
